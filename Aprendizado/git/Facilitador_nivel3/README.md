# Resumo nivel 3

Pode-se configurar o histórico de commits de acordo com a necessidade e preferência 
de cada desenvolvedor, lembrando que isso influenciará na maneira de serem exibidos(git rebase/git merge).

Um template bem elaborado torna-se indispensável em um ambiente colaborativo, para que 
possamos observar determinados tipos de prioridades de um projeto a serem atendidas.

O método KanBan torna-se altamente eficiente no quesito de observabilidade de estado de atividades,
contribuindo para controle de ESTADOS DE CONCLUSÃO de cada atividade.

A utilização de uma pipeline eficiente para revisamento e checagem de código se torna
altamente eficiente no quesito de prever erros e manter um desenvolvimento sólido. 
Ocorrendo da seguinte maneira: Aloca-se a ferramenta eslint nos "scripts" do /package-json/, permitindo que o arquivo YAML (lint.yml) rode os comandos de lint, executando assim a atividade de checagem de código, essa checagem ocorre de acordo com a aplicação das regras presentes no eslint.config.js
